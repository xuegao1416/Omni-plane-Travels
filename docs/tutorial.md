# 上手教程：从零开始你的第一次世界观漫游

> 适用版本：Omniplane Travels v2.5.0（Web / PWA 优先，桌面端紧随）。
> 本教程带你走完「配置 API Key → 创建世界 → 发第一条消息 → 查看 / 回滚变量」完整闭环。预计用时 10–15 分钟。

---

## 0. 开始前你需要什么

- 一个 **OpenAI 兼容的 API**（OpenAI / DeepSeek / Google AI / 自定义端点皆可）。
- 你的 **API Key**（由对应 Provider 提供）。
- 浏览器（推荐桌面 Chrome / Edge / Firefox）。桌面端 Tauri 应用无需配置代理。

> 找不到 Key？去向你的 Provider 控制台申请。本项目**不代管、不代售**任何 Key。

---

## 1. 配置 API Key（含可选 CORS 代理）

### 1.1 打开 API 设置

1. 启动应用后，进入主菜单的 **设置**。
2. 切到 **API 设置** Tab。
3. 在「参数配置」右侧的 Provider 下拉中选择你的服务商：
   - `OpenAI 兼容`（默认）
   - `DeepSeek`
   - `Google AI`
   - `自定义`（任意 OpenAI 兼容端点）

### 1.2 填写四项核心参数

| 字段 | 说明 | 示例 |
|------|------|------|
| **API 端点 (baseUrl)** | Provider 的 `/v1` 地址 | `https://api.openai.com/v1` |
| **API Key** | 你的密钥，**仅存本机加密存储** | `sk-...` |
| **模型名称 (model)** | 想用的模型 | `gpt-4o`、`deepseek-chat` |
| **代理地址（可选）** | 见下方 §1.3 | `https://api-proxy.你的用户名.workers.dev` |

> 可选填温度、topP、topK、maxTokens、上下文大小、流式、思考链等高级参数。

### 1.3 （网页端）可选：CORS 代理

浏览器有安全限制（CORS），部分 Provider 不允许网页直接调用。两种解法：

- **方案 A（推荐桌面用户）**：使用 Tauri 桌面端，原生 HTTP 链路，无需代理。
- **方案 B（网页用户）**：在 **API 设置** 的「代理地址（可选）」填入一个**你自己部署**的 CORS 代理。应用内提供 **代理部署教程**（设置页入口），照着 5 分钟在 Cloudflare Workers 上部署一个即可。代理只做转发、**不存储数据、完全由你控制**。

### 1.4 测试连接

填完后点击 **测试连接**。显示「连接成功」即配置生效。若失败：

- 确认 baseUrl 是否带 `/v1`；
- 确认 Key 未过期、未欠费；
- 网页端若报 CORS，回到 §1.3 配置代理；
- 打开控制台，查看带 `[Proxy]` 标记的紫色折叠日志排错。

## 2. 创建世界（创建向导）

> 应用已内置 6 个世界观（日式校园 / 欲望魔都 / 武侠世界 / 末日废土 / 荒岛求生 / 边境贸易），也可从零自建。

1. 回到主菜单，选择 **创建角色 / 新游戏**，进入 **世界创建向导**（WizardShell）。
2. 向导分步（实际步骤名以界面为准）：
   - **世界浏览 / 世界选择**：挑一个内置世界，或选「自建」。
   - **世界详情**：查看该世界的设定、规则、模块配置。
   - **个人信息**：填写你的出身、职业、阶层等（写入 PlayerState）。
   - **角色经历**：补充角色背景故事（可选，AI 会据此生成更贴合的叙事）。
   - **模块选择 (ModuleSelector)**：勾选本世界启用的游戏系统——`数值属性 / 成长体系 / 生存资源 / 经营资产 / 骰子检定 / 天赋体系`。
   - **确认**：核对后生成世界。
3. 点击 **确认**，引擎走 7 阶段世界创建管线（种子 → 骨架 → 维度 → 一致性 → 深描 → 世界书 → 模块数据），自动产出世界设定与初始变量。

> 想导入别人的世界？支持 **JSON 导入**（世界定义 + 世界书），详见存档管理。

---

## 3. 发第一条消息

进入游戏主屏幕（GameScreen），你会看到三块区域：

- **中间聊天面板 (ChatPanel)**：对话主战场。
- **右侧面板 (RightPanel)**：世界状态、角色属性、待办、变量快照。
- **左侧面板**：角色档案、NPC 网格、笔记本、世界书。

**发出你的第一条行动**：

1. 在底部 **输入区 (InputArea)** 输入你的行动，例如：
   > 「我推开酒馆的木门，环视四周，寻找那个传说中的情报贩子。」
2. 点击发送，AI 实时**流式回复**（可中途点「停止」）。
3. 也可点击 AI 推荐的 `[OPTION]` 选项快速推进。

每条 AI 回复后，引擎会：
- 经 **11 阶段 AI 管线** 维护叙事连贯与记忆；
- 通过独立调用提取 `<UpdateVariable>` 自动更新游戏变量；
- 把当前 **GameState 快照** 附加到这条消息上（供回滚）。

---

## 4. 查看 / 回滚变量面板

### 4.1 打开变量快照面板

1. 在游戏界面打开 **变量快照面板 (VariableSnapshotPanel)**（左侧 / 右侧面板中的「变量」入口）。
2. 面板列出所有快照层级：
   - **当前状态（最新）**——实时 GameState；
   - 每条 AI 消息附带的 **历史快照**（倒序，最多保留首条 + 最近 10 条 + 每 10 条关键帧）。

每个快照可展开查看 JSON 结构，包含 `世界 / 玩家 / 人物档案` 等。

### 4.2 回滚到任意历史节点

1. 在快照列表里选一条历史快照。
2. 点击 **回滚 (Rollback)**，弹出确认框。
3. 确认后，GameState 恢复到该节点，**记忆系统检查点同步恢复**。
4. 此后从该节点继续游玩即可。

> 回滚只改状态，不删消息。你也可以直接「载入」某层快照的 JSON 进行手工微调后再继续。

## 5. 常见问题（FAQ）

**Q：测试连接成功，但发消息卡住不动？**
A：多为流式超时。等待约 120s 会自动 abort 并提示重试；也可手动点「停止」再发。仍卡死请查控制台日志。

**Q：变量没更新 / 数值异常？**
A：确认该世界勾选了对应模块（如数值属性）。变量由 AI 经 `<UpdateVariable>` 写入，若模型不稳定可能漏写——可在变量面板手工编辑 JSON 后「载入」。

**Q：刷新页面后数据还在吗？**
A：在。存档持久化于 IndexedDB，F5 通过 `active_save_id` 自动恢复上次存档。

**Q：Key 安全吗？**
A：Key 仅加密存于本机，只发给你配置的 Provider。详见 [隐私政策](../PRIVACY.md)。

**Q：能离线玩吗？**
A：首版 PWA 不把「离线可用」作为卖点，仅保证可安装。对话需联网调用 LLM。
